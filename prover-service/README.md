# PrivaMargin LTV Prover Service

External Groth16 proof generation service for LTV computations.

## Architecture

```
CRE Workflow → Prover Service → Groth16 Proof → On-chain Verifier
     ↓              ↓                ↓                ↓
  Computes      Generates        Returns         Verifies &
  LTV values    ZK proof         calldata        records result
```

## Why External Prover?

The CRE WASM runtime has performance constraints that make snarkjs proof generation impractical (15-60s per proof). This service runs externally and can be:

- **Cloudflare Worker** (for lightweight proofs)
- **Node.js service** (for production workloads)
- **TEE environment** (for enhanced security)

## Setup

### Prerequisites

- Node.js 18+
- circom 2.1+
- snarkjs 0.7+

```bash
# Install circom
npm install -g circom

# Install snarkjs
npm install -g snarkjs

# Install dependencies
npm install
```

### Compile Circuit

```bash
npm run setup:circuit
```

This will:
1. Download Powers of Tau (trusted setup)
2. Compile the circom circuit
3. Generate proving and verification keys
4. Create Solidity verifier contract

### Local Development

```bash
npm run dev
```

### Deploy to Cloudflare

```bash
npm run deploy
```

## Circuit

The LTV circuit (`circuits/ltv.circom`) verifies:

1. Collateral value computation: `sum(amount[i] * price[i])`
2. Effective collateral: `totalCollateral + unrealizedPnL`
3. LTV formula: `notional / (effectiveCollateral * leverage)`
4. Breach check: `LTV >= threshold`

**Public Inputs:**
- `ltvResultBps` - Computed LTV in basis points
- `thresholdBps` - Threshold in basis points
- `positionIdHash` - Hash of position ID
- `timestamp` - Proof generation time

**Private Inputs:**
- Collateral amounts and prices
- Notional value
- Unrealized PnL
- Leverage ratio

## On-Chain Integration

Deploy the contracts to verify proofs on-chain:

1. Deploy `LTVVerifier.sol` (generated by setup)
2. Deploy `LTVOracle.sol` with verifier address
3. Authorize CRE workflow address as submitter

```solidity
// Check if position is breached
(bool isBreached, uint256 ltvBps, uint256 lastUpdate) =
    ltvOracle.checkPosition(positionIdHash);
```

## Security Considerations

- Proofs are only valid for 5 minutes (freshness check)
- Position ID is hashed to prevent linkability
- On-chain verification is trustless
- External prover can be run in TEE for additional security
